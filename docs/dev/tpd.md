## 文档边界

本文档定义 life-ledger 首版开发过程中的测试策略、测试范围、质量门控和验收标准。

本文档不替代具体测试用例代码：

- API 细节后续以 `docs/dev/api-design.md` 为准。
- SQLite 表结构和索引以后续 `docs/dev/data-model.md` 为准。
- 部署与恢复操作以后续 `docs/dev/deployment.md` 为准。
- 每个开发阶段的测试执行顺序以 `docs/dev/task.md` 为准。

## 1. 测试目标

- 验证 PRD 中 R1-R19 的 P0 需求全部可用，P1 需求至少不阻塞核心流程。
- 验证单二进制部署、`config.toml`、SQLite、内嵌前端和 Caddy 反代架构可以形成闭环。
- 验证账单 Excel 导入导出不会产生部分写入、原始内容泄露或错误数据污染。
- 验证单用户登录、设备会话、CSRF、CORS、安全响应头、审计和文件权限符合安全要求。
- 验证常规个人数据量下核心 API 和页面没有明显性能退化。

## 2. 测试分层

| 层级 | 目标 | 主要覆盖 | 是否必须自动化 |
|------|------|----------|----------------|
| 单元测试 | 验证纯业务规则和边界条件 | 配置校验、字段校验、重复规则、预算计算、Excel 行校验、权限判断 | 是 |
| 集成测试 | 验证模块与 SQLite 的协作 | migration、repository、transaction、审计写入、会话持久化、登录失败限速 | 是 |
| API 测试 | 验证 HTTP 行为 | 认证、CSRF、CRUD、Excel 上传下载、错误响应、安全响应头 | 是 |
| 前端 E2E 冒烟 | 验证核心页面闭环 | 登录、三页导航、基础新增/编辑/删除、刷新路由保持 | 是 |
| 部署冒烟 | 验证真实部署形态 | 单二进制、配置文件、文件权限、Caddy 反代、备份命令 | 首版可手工，发布前必须执行 |
| 安全回归 | 验证安全约束不退化 | Cookie、CSRF、CORS、日志脱敏、审计、设备撤销、限速 | 是 |

## 3. 测试环境

### 3.1 本地自动化环境

- 使用临时目录生成 `config.toml`、数据目录和 SQLite 数据库。
- 每个集成测试使用独立数据库文件，测试结束后删除临时目录。
- 测试中不得依赖开发者机器上的真实 `config.toml` 或真实数据库。
- 测试中不得写入仓库目录外的固定路径，除非测试明确覆盖备份命令并使用临时目录。

### 3.2 前端 E2E 环境

- 使用测试专用配置启动后端。
- 使用测试数据库预置最小数据。
- 浏览器测试只访问本地地址。
- E2E 测试必须覆盖刷新 `/important-dates`、`/transactions`、`/decisions` 后仍能进入对应页面。

### 3.3 部署冒烟环境

- 使用 Linux 环境执行。
- 应用监听 `127.0.0.1:<port>`。
- Caddy 反向代理到应用端口。
- `config.toml` 权限为 `600`，数据目录权限为 `700`，数据库文件权限为 `600`。

## 4. 测试数据原则

- 测试数据只使用虚构个人信息，不使用真实账单、真实证件日期或真实决策内容。
- Excel 导入测试文件必须覆盖合法文件、非法表头、空必填列、非法枚举、非法金额、超行数和错误行混合场景。
- 安全测试不得把明文密码、`password_hash`、`session_secret`、Cookie、session token、CSRF token 或 Excel 原始内容输出到日志断言快照中。
- 审计测试只断言资源类型、资源 ID、事件类型、操作结果和脱敏失败原因，不断言完整业务内容。

## 6. 功能测试范围

### 6.1 服务启动和配置

- 合法 `config.toml` 可以启动服务。
- `init-config` 可以创建 `config.toml`、`data/` 和 `backups/`，并拒绝覆盖已有配置。
- 缺失配置文件、TOML 格式错误、必填项缺失时启动失败。
- `session_secret` 缺失或短于 32 字节时启动失败。
- 配置中出现明文密码字段时启动失败。
- 数据目录不存在时按需求创建或返回明确错误。

### 6.2 SQLite 和 migration

- 数据库文件不存在时自动创建并执行全部 migration。
- 已执行过的 migration 不重复执行。
- 新增 migration 只执行未执行部分。
- migration 失败时服务启动失败，并输出失败编号。
- 批量写入失败时 transaction 回滚。

### 6.3 三个一级页面

- `/` 进入 `/important-dates`。
- `/important-dates`、`/transactions`、`/decisions` 刷新后页面不丢失。
- 一级导航切换时内容和高亮同步。
- 未登录访问页面时进入登录页。

### 6.4 重要日期

- 新增、查看、编辑、删除重要日期。
- 必填字段缺失时拒绝保存。
- 重复规则为空时按不重复处理。
- 重复规则不是 `不重复`、`每年`、`每月`、`每周` 时拒绝保存。

### 6.5 账单和预算

- 新增、查看、编辑、删除账单。
- 金额为空、为零或格式非法时拒绝保存。
- `类型` 只允许 `收入` 或 `支出`。
- `计入收支=否` 的账单不参与收入、支出、余额统计。
- `计入预算=是` 仅对支出账单生效。
- 月份 + 分类预算可以创建、更新、删除。
- 修改或删除参与预算的账单后，预算使用金额重新计算。
- 超支分类展示超支状态。

### 6.6 决策

- 新增、查看、编辑、删除决策。
- 候选方案保存为结构化数据。
- 未做最终选择时展示为进行中。
- 复盘日期到达后展示为待复盘。
- 完成复盘并归档后进入已归档分组。

### 6.7 标签

- 重要日期、账单、决策可以添加标签。
- 标签可以在列表和详情中展示。
- 按标签筛选时只返回关联记录。
- 删除业务记录时不应留下无法清理的错误关联。

## 7. Excel 测试范围

### 7.1 模板下载

- 模板文件格式为 `.xlsx`。
- 必填列表头为：日期、时间、类型、金额、分类、计入收支、计入预算、所属账本。
- 选填列表头为：对象、账户、标签、备注。

### 7.2 导入校验

- 非 `.xlsx` 文件拒绝导入。
- 超过 5MB 拒绝导入。
- 超过 5000 行拒绝导入。
- 多工作表文件只读取第一个工作表。
- 表头与模板不一致时拒绝导入。
- 必填列空值时返回行号、列名和原因。
- 类型不是 `收入` 或 `支出` 时拒绝导入。
- `计入收支` 或 `计入预算` 不是 `是` 或 `否` 时拒绝导入。
- 存在任一错误行时不写入任何账单。

### 7.3 导入写入

- 全部合法时在一个 SQLite transaction 中写入。
- 写入过程中失败时 transaction 回滚。
- 成功或失败均写入审计事件。
- 错误响应不回显整行原始内容。

### 7.4 导出

- 导出文件可被 Excel 打开。
- 导出表头与模板字段一致。
- 导出结果符合筛选条件。
- 导出接口需要登录态。

## 8. 安全测试范围

### 8.1 登录和会话

- 未登录请求受保护 API 返回 401。
- 正确用户名和密码登录成功。
- 错误凭据不创建登录态。
- 登录成功后 Cookie 为 HttpOnly；HTTPS 部署时为 Secure；SameSite 使用 Lax。
- SQLite 只保存 session token 哈希。
- 7 天有效期固定，不因访问而延长。
- 过期、撤销或不存在的 token 被拒绝并清除 Cookie。

### 8.2 设备管理

- 登录成功后创建设备会话。
- 设备记录包含设备名称、User-Agent、首次登录时间、最后访问时间、最后访问 IP、过期时间和撤销状态。
- 设备管理展示当前有效设备、过期设备和已撤销设备。
- 撤销非当前设备后，该设备后续请求失效。
- 撤销当前设备后，当前浏览器退出登录。

### 8.3 登录失败限速

- 同一用户名或同一 IP 在 10 分钟内连续失败 5 次后锁定。
- 锁定期内不校验密码，直接拒绝。
- 15 分钟后允许重新尝试。
- 登录成功后清除用户名和 IP 的活跃失败计数。
- 服务重启后未过期锁定仍然生效。

### 8.4 CSRF

- 写操作 API 缺少 `X-CSRF-Token` 返回 403。
- 写操作 API 携带错误 token 返回 403。
- 写操作 API 携带正确 token 且登录态有效时继续处理。
- `GET`、`HEAD`、`OPTIONS` 不要求 CSRF token。
- 退出登录或撤销设备后，对应 CSRF token 失效。

### 8.5 CORS 和响应头

- 任意 API 响应不得包含 `Access-Control-Allow-Origin: *`。
- 第三方 Origin 的跨域请求不能读取受保护 API 响应。
- 页面和 API 响应设置 `X-Frame-Options: DENY`。
- 页面和 API 响应设置 `X-Content-Type-Options: nosniff`。
- 页面和 API 响应设置 `Referrer-Policy: same-origin`。

### 8.6 可信反代 IP

- 请求来自可信反代地址时，使用 `X-Forwarded-For` 中第一个 IP。
- 请求不来自可信反代地址时，忽略客户端伪造的 `X-Forwarded-For` 和 `X-Real-IP`。
- 登录设备、登录失败限速和审计使用同一套解析后的客户端 IP。

### 8.7 日志和审计

- 登录成功、登录失败、退出、设备创建、设备撤销写入审计。
- Excel 导入成功和失败写入审计。
- 删除账单、重要日期或决策写入审计。
- 配置文件、数据目录或数据库权限检查失败时，数据库不可用前输出脱敏错误并非零退出。
- 日志不得打印明文密码、`password_hash`、`session_secret`、session token、Cookie、CSRF token 或 Excel 原始内容。
- 审计事件不得记录完整业务内容。

## 9. 备份和恢复测试范围

- `./life-ledger backup` 成功生成备份包。
- 备份包包含 SQLite 数据库、`config.toml` 和 `backup-meta.json`。
- `backup-meta.json` 包含应用版本、备份时间和数据库 schema 版本。
- 备份无法读取数据库或配置文件时非零退出。
- `backup` 命令不启动 HTTP 服务、不修改数据库。
- 按 `docs/dev/deployment.md` 的恢复步骤停止服务、替换配置和数据目录、修正权限、重新启动后，数据可读。

## 10. 性能测试范围

- 5000 条账单以内，常规列表 API P95 小于 300ms。
- 5000 条账单以内，账单页收入、支出、余额和预算使用统计能在可接受时间内返回。
- Excel 导入 5000 行以内给出明确成功或失败结果。
- 列表接口必须分页，默认 `page_size=50`，最大 `page_size=200`。
- 大于最大分页大小的请求应被拒绝或截断到最大值，行为以后续 API 文档为准。

## 11. 测试门控命令

后端：

```bash
test -z "$(gofmt -l ./cmd ./internal)"
go vet ./...
go test ./...
go test -race ./...
make build
```

前端：

```bash
npm run typecheck
npm run lint
npm run build
npm run test:e2e
```

发布前完整门控：

```bash
go vet ./... && go test ./... && npm run typecheck && npm run lint && npm run build && make build
```

说明：`make build` 是发布二进制的标准构建入口，输出 `bin/life-ledger`。

## 12. 发布准入标准

首版发布前必须满足：

- 所有 P0 需求的自动化或部署冒烟测试通过。
- 账单 Excel 导入失败场景验证不能产生部分写入。
- 安全测试中认证、CSRF、CORS、响应头、日志脱敏、审计、设备撤销和登录失败限速全部通过。
- SQLite migration 测试通过，migration 失败能阻止启动。
- 备份命令测试通过，恢复流程至少完成一次手工演练。
- 前端 E2E 冒烟覆盖登录、三页导航和每个核心页面的一条新增/删除链路。
- 发布前完整门控命令通过。
