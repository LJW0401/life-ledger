# UI 信息架构

life-ledger 前端采用 React 单页应用。首版只保留三个一级页面，每个一级页面对应一条路由；页面内部的小视图使用标签、分段控件或局部筛选切换。

## 页面结构

```text
/
  重定向到 /important-dates

/important-dates
  日期页

/transactions
  账单页

/decisions
  决策页
```

## 导航结构

一级导航只包含三个入口：

- 日期
- 账单
- 决策

桌面端使用左侧导航或顶部导航均可，具体视觉设计后续确定。移动端优先使用底部导航，保证三个核心入口可以单手切换。

## 页面拆分原则

- 一级业务模块使用路由，不放在同一个页面里用全局 Tab 切换。
- 页面内部的不同视图使用局部切换，例如列表、日历、统计、导入导出。
- 浏览器刷新、前进、后退必须能保持当前一级页面。
- 首版不实现提醒能力，也不设置独立仪表盘、提醒中心、分析页或设置页。

## 首版范围

首版页面职责如下：

- 日期页：管理重要日期和重复规则。
- 账单页：管理收入支出流水，并支持 Excel 导入导出。
- 决策页：记录关键决策、候选方案、决策结果和复盘信息。
